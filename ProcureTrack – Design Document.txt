ProcureTrack – Design Document
Procurement & Onboarding Workflow Management System
________________


1. Purpose of This Design Document
This Design Document translates the Product Requirements Document (PRD) into visual, structural, and interaction-level specifications.
The goal is to:
* Define screen structure and layout

* Establish UI patterns and components

* Ensure role-based clarity

* Guide future Figma / Gemini / implementation work

* Avoid ambiguity during development

This document focuses on MVP (Term 1), with clear extension points for Term 2.
________________


2. Design Inspiration & Direction
Design Inspiration Sources
   * Dribbble: Modern SaaS dashboards (Notion, Linear-style layouts)

   * Mobbin: Enterprise onboarding flows (BambooHR, Rippling-style UX)

Design Philosophy
      * Clean, minimal, enterprise-first

      * Checklist-driven UX

      * Low cognitive load

      * Data visibility for Admins

      * Task clarity for Employees/Vendors

Visual Style
Element
	Direction
	Color Palette
	Neutral base (gray/white) + single accent (blue or green)
	Typography
	Sans-serif (Inter / Roboto style)
	Layout
	Card-based, grid layout
	Icons
	Line icons (Lucide / Material Icons)
	Motion
	Minimal (progress bars, hover states only)
	________________


3. User Roles & Access Model
Supported Roles (MVP)
         1. Admin

         2. HR / Procurement Staff

         3. Employee

         4. Vendor

Each role sees only relevant workflows and actions.

4. Information Architecture (IA)
Global Navigation
- Dashboard

- Workflows (Admin only)

Navigation adapts based on role.

5. Core Screens & Wireframe-Level Specs
________________


5.1 Authentication Screens
Login Screen
Purpose: Secure role-based entry
Components
            * Email field

            * Password field

            * Role inferred from backend

            * Login CTA

            * Error messaging

UX Notes
               * No role selection on login (pre-assigned)

               * Clean, distraction-free screen

________________


5.2 Admin Dashboard
Purpose
High-level visibility into onboarding progress.
Layout
Top Section
                  * KPI cards:

                     * Total Active Onboardings

                     * Completed (%)

                     * Pending Tasks

                     * Overdue Tasks

Middle Section
                        * Table: Active Onboardings

                           * Name

                           * Role (Employee/Vendor)

                           * Workflow

                           * Progress %

                           * Status

Bottom Section
                              * Recent activity feed

________________


5.3 Workflow Builder (Admin – MVP Focus)
Purpose
Create reusable onboarding workflows.
Layout
Left Panel
                                 * Workflow list

                                 * Create new workflow CTA

Main Panel
                                    * Workflow Name

                                    * Workflow Description

                                    * Task List (stacked cards)

Task Card Components
                                       * Task Name

                                       * Task Type:

                                          * Checkbox

                                          * Document Upload

                                             * Optional Notes

                                             * Required / Optional toggle

UX Notes
                                                * Simple linear workflows (no branching in MVP)

                                                * Drag & drop optional (stretch goal)

________________


5.4 Assign Workflow Screen
Purpose
Assign workflows to users.
Components
                                                   * User selector

                                                   * Workflow selector

                                                   * Assignment date

                                                   * Due date (optional)

                                                   * Assign CTA

Result
                                                      * User instantly sees tasks in their dashboard

________________


5.5 Employee / Vendor Dashboard
Purpose
Task-focused, zero clutter.
Layout
Top
                                                         * Assigned Workflow Name

                                                         * Progress bar (e.g., 60% complete)

Task List
                                                            * Task name

                                                            * Status (Pending / Completed)

                                                            * Due date (if any)

                                                            * Upload button (if document required)
